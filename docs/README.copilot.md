# Superpowers for GitHub Copilot

Complete guide for using Superpowers with GitHub Copilot agent mode in VS Code.

## Quick Install

Tell Copilot:

```
Fetch and follow instructions from https://raw.githubusercontent.com/obra/superpowers/refs/heads/main/.copilot/INSTALL.md
```

## Manual Installation

### Prerequisites

- VS Code with GitHub Copilot extension
- GitHub Copilot subscription (Individual, Business, or Enterprise)
- Copilot agent mode enabled (VS Code 1.99+ or Insiders)
- Git

### Global Install

1. Clone the repo:
   ```bash
   git clone https://github.com/obra/superpowers.git ~/.copilot-superpowers
   ```

2. Add to VS Code Settings (JSON):
   ```json
   {
     "github.copilot.chat.codeGeneration.instructions": [
       { "file": "~/.copilot-superpowers/.github/copilot-instructions.md" }
     ]
   }
   ```

3. Restart VS Code.

### Per-Project Install

1. Add superpowers as a submodule:
   ```bash
   git submodule add https://github.com/obra/superpowers.git .superpowers
   ```

2. Generate the instructions file with per-project paths:
   ```bash
   mkdir -p .github
   bash .superpowers/scripts/generate-copilot-instructions.sh --prefix .superpowers
   ```

3. Commit `.github/copilot-instructions.md` so your team benefits automatically.

### Windows

```powershell
git clone https://github.com/obra/superpowers.git "$env:USERPROFILE\.copilot-superpowers"
```

Use the full Windows path in VS Code settings:
```json
{
  "github.copilot.chat.codeGeneration.instructions": [
    { "file": "C:\\Users\\YourName\\.copilot-superpowers\\.github\\copilot-instructions.md" }
  ]
}
```

## How It Works

Copilot agent mode reads `.github/copilot-instructions.md` at the start of every chat session and injects it as custom context. Superpowers uses this mechanism to bootstrap the skills system — the instructions file contains Copilot-adapted skill workflow content plus a catalog of all available skills.

Unlike Claude Code or Cursor (which use hooks and plugin systems), Copilot relies on a static instructions file. This means:

- **No hooks** — the instructions are loaded once per session, not dynamically
- **No native skill tool** — skills are loaded by reading their SKILL.md files directly
- **No subagents** — multi-agent skills fall back to single-session execution

### Skill Loading

When a skill applies to your task, the agent reads the SKILL.md file directly:

```
~/.copilot-superpowers/skills/<skill-name>/SKILL.md
```

For per-project installs: `.superpowers/skills/<skill-name>/SKILL.md`. The generated instructions file contains absolute paths matching your install method.

### Tool Mapping

Skills are written using Claude Code tool names. In Copilot agent mode:

| Skill references | Copilot equivalent |
|---|---|
| `Read`, `Write`, `Edit` | Built-in file tools |
| `Bash` | Run in terminal |
| `Grep` / `Glob` | Workspace search or terminal commands |
| `Skill` tool | Read the SKILL.md file directly |
| `Task` tool (subagents) | Not supported — use `executing-plans` |
| `TodoWrite` | Use markdown checklists |

## How to Invoke Skills

Skills activate automatically when the agent recognizes a matching task. You can also invoke them explicitly. Here are the ways to trigger a skill:

**Natural language (recommended):**
```
Use the brainstorming skill to help me design this feature
Debug this systematically
Use TDD for this bugfix
```

**Direct file reference:**
```
Read ~/.copilot-superpowers/skills/brainstorming/SKILL.md and follow it
```

**For per-project installs**, paths use `.superpowers/` instead:
```
Read .superpowers/skills/test-driven-development/SKILL.md and follow it
```

The agent knows the full skill catalog from `.github/copilot-instructions.md`, so natural language triggers work best — you don't need to memorize file paths.

### Personal Skills

Create your own skills alongside superpowers:

```bash
mkdir -p ~/.copilot-superpowers/skills/my-skill
```

Create `~/.copilot-superpowers/skills/my-skill/SKILL.md`:

```markdown
---
name: my-skill
description: Use when [condition] - [what it does]
---

# My Skill

[Your skill content here]
```

Add the skill to the catalog section in `.github/copilot-instructions.md` so the agent knows it exists.

## Platform Limitations

| Feature | Status |
|---|---|
| Skill discovery & invocation | Via file reading (no native tool) |
| Session start hooks | Not supported (static instructions only) |
| Subagent dispatch | Not supported (falls back to executing-plans) |
| Parallel agents | Not supported |
| Web search / fetch | Not available in agent mode |
| Task tracking (TodoWrite) | Not supported (use markdown checklists) |

## How It Stays In Sync

The `.github/copilot-instructions.md` file is auto-generated by `scripts/generate-copilot-instructions.sh`. This script produces Copilot-native instructions (not a raw dump of the `using-superpowers` skill) with:
- Copilot-adapted skill workflow (file reading instead of tool invocation)
- The Copilot tool mapping from `references/copilot-tools.md`
- An auto-generated catalog of all available skills with absolute paths

The script accepts `--prefix <path>` to control skill paths (default: `~/.copilot-superpowers`).

When you `git pull`, the pre-built file updates automatically. To regenerate after adding custom skills:
```bash
bash scripts/generate-copilot-instructions.sh
```

## Updating

```bash
cd ~/.copilot-superpowers && git pull
```

For per-project installs, update the submodule and regenerate:
```bash
git submodule update --remote .superpowers
bash .superpowers/scripts/generate-copilot-instructions.sh --prefix .superpowers
```

## Troubleshooting

### Skills not recognized

1. Verify the instructions file is loaded: ask Copilot "Do you have superpowers?"
2. Check VS Code settings — ensure the file path in `codeGeneration.instructions` is correct
3. Restart VS Code after config changes

### Agent mode not available

1. Copilot agent mode requires VS Code 1.99+ or VS Code Insiders
2. Ensure agent mode is enabled in Copilot settings
3. Check your Copilot subscription is active

### Skills not loading

1. Verify the skills directory exists: `ls ~/.copilot-superpowers/skills/`
2. Check skill paths in the instructions file match your installation
3. The agent reads skills by file path — ensure the path is accessible from your workspace
