# Issue Tracker

This repository uses GitHub Issues as the canonical issue tracker.

Use GitHub Issues for:

- user-facing bugs
- feature requests
- repo maintenance work
- work that needs triage, assignment, or backlog tracking

Keep issue titles short and outcome-focused. Write the problem in the issue body before suggesting a fix.
