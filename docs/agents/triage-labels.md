# Triage Labels

Use a small, stable GitHub label set:

- `type:bug` - something is broken or incorrect
- `type:enhancement` - an improvement or new capability
- `priority:high` - needs attention soon
- `status:blocked` - blocked on a dependency or decision
- `area:skills` - affects a skill or skill workflow
- `area:docs` - affects documentation, guides, or references
- `area:tooling` - affects scripts, hooks, or workspace tooling

Label rules:

- Prefer one `type:` label per issue.
- Use `priority:` only when urgency changes the order of work.
- Use `status:` labels only for active blockers.
- Use `area:` labels to route ownership and reading context.
