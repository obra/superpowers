milestone

## Admission Rule
Use this category for larger complete changes that form a meaningful feature, architecture adjustment, or workflow closure.

## Notes
- Record only closed, reviewable change units.
- Prefer milestone when the main value is new capability, system-level closure, or a meaningful end-to-end improvement.
- Do not use this category for debugging-only closures or maintainability-first structural cleanups.

## Entry Focus
- What feature, architecture, or workflow closure was completed
- Why the change unit matters to future readers
- Which commits together represent the completed unit
