# debug

## Category
debug

## Admission Rule
Use this category for complete debugging closures where the problem is identified, fixed, and verified.

## Notes
- Record only closed debugging loops, not ongoing investigation.
- Prefer debug when the main value is preserving the failure mode, root cause, fix, and verification outcome.
- Do not use this category for new feature delivery or broad structural cleanup whose main value is maintainability.

## Entry Focus
- What symptom or regression was observed
- What root cause was confirmed
- How the fix was verified and why it closes the issue
