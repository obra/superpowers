# `docs/project_notes/bugs.md` Template

Use for recurring failures, expensive rediscovery, and short prevention notes.

```markdown
# Bugs

- YYYY-MM-DD: <short recurring bug summary>
  Root cause: <one sentence>
  Fix: <one sentence>
  Prevention / verification: <one sentence>
  Source: `<approved artifact or stable repo doc>`
```

Maintenance:

- keep only recurring or high-cost bugs
- prune stale one-off noise
- backlink to authoritative evidence when available
