# `docs/project_notes/issues.md` Template

Use for durable cross-session breadcrumbs about tickets, PRs, plans, or evidence artifacts.

```markdown
# Issues

- YYYY-MM-DD: <ticket, PR, plan, or evidence reference>
  What changed: <one sentence>
  Source: `<ticket link, plan path, review artifact, or evidence path>`
```

Maintenance:

- keep breadcrumbs short and durable
- do not track active checklists, status boards, or day-by-day execution progress here
- prune entries that no longer add cross-session value
