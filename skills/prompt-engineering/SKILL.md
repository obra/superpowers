# 高性能プロンプト生成

ユーザーの目的を理解し、Claude・GPT・Geminiなど各モデルに最適化された高性能プロンプトを設計する。

<HARD-GATE>
プロンプトを出力する前に、必ず以下のヒアリングを完了すること。情報が不足したまま生成しない。
</HARD-GATE>

## プロセス

### STEP 1: ヒアリング（1問ずつ）

以下を順番に1問ずつ確認する：

1. **目的** — このプロンプトで何をしたいか？（例：文章生成、分類、要約、コード生成）
2. **対象モデル** — Claude / GPT-4o / Gemini / その他？
3. **出力形式** — 文章？箇条書き？JSON？長さは？
4. **制約・禁止事項** — やってほしくないことはあるか？
5. **具体例** — 理想の入出力サンプルはあるか？

### STEP 2: プロンプト設計

ヒアリング完了後、以下の技法を組み合わせて設計する：

#### 必須要素チェックリスト

- [ ] **役割設定** (Role) — `あなたは〇〇の専門家です`
- [ ] **コンテキスト** — 背景・前提条件を明示
- [ ] **タスク定義** — 何をするか1文で明確に
- [ ] **出力形式指定** — フォーマット・長さ・構造
- [ ] **制約条件** — してはいけないこと
- [ ] **Few-shot例** — 入出力の具体例（2-3個）
- [ ] **思考促進** — 複雑なタスクには `Step by step で考えてください`

#### 高性能化テクニック

**明確性の原則：**
```
❌ 「良い文章を書いて」
✅ 「500字以内で、中学生でも理解できるよう、
    具体的な例を1つ使って説明してください」
```

**Chain of Thought（複雑なタスク用）：**
```
まず〇〇を分析し、次に△△を検討した上で、
最終的に□□という形式で回答してください。
```

**出力構造化：**
```
以下の形式で出力してください：
【タイトル】
【要約】（100字以内）
【詳細】
【結論】
```

**ネガティブプロンプト（禁止事項）：**
```
以下は絶対に含めないでください：
- 曖昧な表現（「場合によっては」など）
- 免責事項
- 不必要な前置き
```

### STEP 3: プロンプト出力

完成したプロンプトをコードブロックで提示する：

```
---プロンプト開始---

[完成プロンプト本文]

---プロンプト終了---
```

### STEP 4: 改善提案

プロンプトの後に以下を提示する：

1. **このプロンプトの強み** — なぜ高性能か
2. **バリエーション案** — 用途別に2-3パターン
3. **チューニングポイント** — 調整できる変数

## プロンプトテンプレート集

### 汎用テンプレート
```
あなたは[役割]の専門家です。

[コンテキスト・背景]

## タスク
[具体的な指示]

## 出力形式
[フォーマット指定]

## 制約
- [禁止事項1]
- [禁止事項2]

## 例
入力：[サンプル入力]
出力：[サンプル出力]
```

### 分析・要約用
```
以下の[コンテンツ種別]を分析してください。

[入力テキスト]

分析観点：
1. [観点1]
2. [観点2]

出力形式：
- 要約（3行以内）
- キーポイント（箇条書き5点）
- 推奨アクション（具体的に）
```

### コード生成用
```
あなたは[言語]の上級エンジニアです。

要件：[機能説明]

制約：
- [言語バージョン]
- [使用可能ライブラリ]
- エラーハンドリングを含める
- コメントを日本語で記述

出力：動作するコードのみ。説明不要。
```

## 品質基準

生成したプロンプトは以下を満たすこと：

| 基準 | チェック項目 |
|------|------------|
| 明確性 | 曖昧な指示がない |
| 完全性 | 必要な情報がすべて含まれる |
| 効率性 | 不要なトークンを使わない |
| 再現性 | 同じ入力で安定した出力が得られる |
| 適応性 | 対象モデルの特性を活かしている |
