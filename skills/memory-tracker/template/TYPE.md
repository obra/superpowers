# Memory Categories

## Type
milestone

### Use When
- The work closes a meaningful feature, architecture, or workflow direction.
- The most important thing to remember is why this direction was chosen and what larger shape it establishes.

### Avoid When
- The main value is debugging evidence or root-cause closure.
- The main value is structural cleanup without a new product or workflow direction.

## Type
debug

### Use When
- The work closes a concrete failure loop with confirmed symptom, root cause, and verification.
- Future readers mainly need to remember why the issue happened and why the closure is trustworthy.

### Avoid When
- The work is mainly about delivering a new capability or workflow.
- The work is mainly about maintainability or architectural cleanup.

## Type
refactor

### Use When
- The work closes a meaningful structural cleanup or boundary improvement.
- Future readers mainly need to remember why the old structure was a burden and what new shape should be preserved.

### Avoid When
- The main value is a user-facing capability or workflow closure.
- The main value is fixing a concrete failure mode.
