# 대화형 학습 회고 (Day 3)

3일간의 학습 여정을 대화형으로 돌아보고, 사용자의 답변을 기반으로 evaluation.md와 retrospective.md를 작성한다.

## 시작 절차

1. `current_run_path`가 없으면 사용자에게 요청한다.
2. `current-run.md`를 Read하여 `run_dir`, `topic` 추출한다.
3. 가능한 학습 산출물 파일들을 Read한다:
   - `{run_dir}/day1/terms.md` (필수)
   - `{run_dir}/day1/quiz.md` (점수 확인)
   - `{run_dir}/day2/oss-analysis.md`
   - `{run_dir}/day2/practice-review.md` (있으면)
   - `{run_dir}/day3/project-spec.md`
   - `{run_dir}/day3/code-review.md` (있으면)
4. 회고 스타일을 선택받는다:

```
📝 {TOPIC} 3일 학습 회고

회고 방식을 선택해주세요:
A) KPT — Keep / Problem / Try (기본 추천)
B) 4L  — Liked / Learned / Lacked / Longed for
C) SSC — Start / Stop / Continue
```

## 회고 진행 루프

수집한 학습 데이터를 바탕으로 질문을 준비하되, 한 번에 하나씩 질문한다.

### 공통 도입 (항상 먼저)

```
3일 학습을 한 줄로 요약한다면?
```

### KPT 모드 질문 흐름

```
[Keep] 3일 동안 효과적이었던 학습 방법이나 습관이 있었나요?

[Keep] (답변 후) 다음 스터디에서도 유지하고 싶은 것은?

[Problem] 가장 어려웠거나 이해가 안 됐던 개념은 무엇인가요?

[Problem] (답변 후) 그 어려움의 원인이 뭐라고 생각하나요?

[Try] 다음 스터디에서 다르게 해보고 싶은 것이 있다면?

[Try] (답변 후) 다음에 공부하고 싶은 주제가 있나요?
```

### 4L 모드 질문 흐름

```
[Liked] 3일 중 가장 재미있었던 순간은?

[Learned] 가장 큰 깨달음 한 가지를 꼽는다면?

[Lacked] 부족하다고 느낀 점이나 더 깊이 공부하고 싶었던 부분은?

[Longed for] 학습 환경이나 자료에서 아쉬웠던 점은?
```

### 평가 질문 (공통, 회고 스타일 무관)

```
프로젝트 구현에서 가장 잘 됐다고 생각하는 부분은?

반대로 아쉬웠던 구현 부분은?

스스로 점수를 매긴다면? (1~5점)
```

### 진행 규칙

- 사용자 답변이 짧으면 "조금 더 구체적으로 말해줄 수 있어요?" 한 번 정도 추가 질문한다.
- 모든 질문이 끝나면 바로 문서 작성으로 넘어간다 (사용자 승인 불필요).
- 회고 중 수집한 답변을 기반으로 문서를 작성하므로 추가 질문 없이 작성 가능해야 한다.

## 문서 작성

모든 질문 완료 후:

```
✏️ 회고 내용을 정리해서 파일로 저장할게요.
- {run_dir}/day3/evaluation.md
- {run_dir}/day3/retrospective.md
```

- `plugin-study-retrospective-guide` agent의 출력 형식을 따라 두 파일을 Write한다.
- 대화에서 나온 답변을 직접 인용/요약하여 파일을 채운다.
- 메트릭(quiz 점수, LOC 등)은 학습 파일에서 자동 추출한다.


## 핵심 원칙

- 질문은 한 번에 하나씩만 한다.
- 분석적 질문보다 경험 기반 질문을 우선한다 ("느꼈나요?" > "분석하면?").
- 긍정적이고 건설적인 톤을 유지한다.
- 사용자가 "건너뛰기"라고 하면 해당 질문을 생략하고 다음으로 넘어간다.
