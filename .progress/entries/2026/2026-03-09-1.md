## Title
Make progress bootstrap initialization deterministic

## Background / Issue
A reviewer noted that `progress-bootstrap` required an `Entry Template` section in `.progress/PROGRESS.md` without specifying its exact markdown, which made initialization non-deterministic.

## Actions / Outcome
- Approach 1: Ran independent baseline checks against the current `skills/progress-bootstrap/SKILL.md` -> both checks reported the `Entry Template` body was unspecified.
- Approach 2: Considered switching bootstrap to copy a concrete template source -> rejected because inlining the exact file body keeps the bootstrap contract self-contained.
- Final approach: Inlined the complete `.progress/PROGRESS.md` payload in `skills/progress-bootstrap/SKILL.md` and added wording to preserve exact line order, blank lines, LF endings, and trailing newline -> bootstrap output is now deterministic.

## Lessons / Refinements
- Initialization skills should embed exact file content when deterministic output is required, not shorthand section names.
- Independent readbacks are useful for testing skill clarity because they reveal ambiguities that the author may gloss over.
