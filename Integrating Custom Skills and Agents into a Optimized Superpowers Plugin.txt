Plan for Integrating Custom Skills and Agents into the Optimized Superpowers Fork
The following plan assumes you are working inside this already-forked and optimized repository (a derivative of https://github.com/obra/superpowers). The goal is to integrate your existing custom skills and agents from `custom-skills-and-agents/` into the core Superpowers workflow so that everything runs as a single, unified agentic framework. We preserve your workflow optimizations (see `docs/plans/2026-03-05-agent-workflow-optimization.md`) while wiring in specialized skills for planning, implementation, testing, and review.
The integration will promote greater results through orchestrated specialization: for instance, invoking your `security-reviewer` during review phases or `prompt-optimizer` for initial refinement. However, not all items are fully compatible without adaptation; `omit-assistant-history` poses a conflict with Superpowers' reliance on session-aware task history and subagent continuity, and is recommended to remain separate (details below).

Step 1: Treat This Repo as the Base

- Ensure this fork stays reasonably up to date with upstream `obra/superpowers` as needed (via a standard `upstream` remote and occasional merges).
- Do not create a separate “optimized” repository; all optimization and integration work happens directly in this repo.
- Confirm the core skills and optimization plan are present and aligned (`skills/`, `docs/plans/2026-03-05-agent-workflow-optimization.md`, `.claude-plugin/`, `.cursor-plugin/`, etc.).

Update Documentation (High-Level):
- Modify `README.md` to briefly describe this fork as a “unified Superpowers-based framework with additional specialized skills and agents” and mention the major additions (security review, senior engineer, frontend craftsmanship, prompt optimization, testing specialist).

Step 2: Adapt and Integrate Custom Skills and Agents
Each custom file must be converted to Superpowers' format: place content in `skills/<slug>/SKILL.md`, refine frontmatter for compatibility, and, where helpful, add short notes tying the skill to Superpowers workflow phases. Preserve core content but keep any added integration notes concise to respect token and context budgets from your optimization plan.

Synergy: Hook into `skills/brainstorming/SKILL.md` (and optionally `skills/writing-plans/SKILL.md`): “For especially vague or multi-part requests, consider using `prompt-optimizer` once before refining the design.”

omit-assistant-history (SKILL.md) (Not Compatible; Keep Separate):
Source: `custom-skills-and-agents/custom-skills/omit-assistant-history.md`
Reason: Superpowers relies on persistent task history, subagent continuity, and session-aware progress (e.g., dependency tracking and multi-phase plans). Omitting assistant history can break gating, cause lost context in multi-phase workflows, or lead to redundant task recreation. Empirical benefits (reduced pollution) are outweighed by these risks in this specific agentic setup.
Recommendation: Maintain this as a standalone skill (e.g., via direct load in Claude Code or Cursor) for ad-hoc sessions that do not use the full Superpowers workflow. Do not wire it into core Superpowers skills or hooks.


Step 3: Orchestrate Workflow Synergy

Modify Core Skills:
- In `skills/brainstorming/SKILL.md`, add brief, high-signal mentions of `prompt-optimizer` and `claude-md-creator` as optional helpers (for vague prompts and missing AGENTS/CLAUDE docs).
- In `skills/writing-plans/SKILL.md`, mention `prompt-optimizer` as a pre-step for complex or ambiguous features, and `claude-md-creator` for missing context files.
- In `skills/executing-plans/SKILL.md` and `skills/subagent-driven-development/SKILL.md`, reference `senior-engineer` for complex/architectural tasks and `frontend-craftmanship` for frontend/UI work.
- In `skills/test-driven-development/SKILL.md`, reference `testing-specialist` as an optional advanced-testing advisor.
- In `skills/requesting-code-review/SKILL.md` (and optionally `skills/receiving-code-review/SKILL.md`), reference `security-reviewer` as part of the review flow, especially for security-sensitive changes.

Respect Optimization and Context Hygiene:
- Keep all added text short and operational; do not reintroduce long boilerplate.
- Ensure integration guidance aligns with `adaptive-workflow-selector` and `context-management` (e.g., avoid forcing heavy skills in lightweight flows).

Step 4: Testing and Iteration

- Install this fork into your IDE (Cursor and/or Claude Code) and run representative workflows (e.g., “Implement a secure frontend login page,” “Refactor a complex backend module with tests”). Observe when and how the new skills are suggested or invoked.
- Try ambiguous prompts to exercise `prompt-optimizer` and confirm it improves clarity without bloating context.
- Deliberately introduce security and testing concerns to validate `security-reviewer` and `testing-specialist` usage.
- If any skill feels over-eager or under-used, revise its mentions in the core SKILL files (or your agent configuration) accordingly.

Step 5: Publish and Maintain

- Commit your changes with descriptive messages and, when stable, create a tagged release (e.g., `v1.0-unified-superpowers`).
- Optionally publish via a marketplace or internal distribution mechanism, clearly marking it as your “optimized, unified Superpowers” fork.
- Periodically sync with upstream `obra/superpowers`, re-running key workflows after significant updates to ensure compatibility.

This integration yields a combined framework with enhanced specialization, reducing errors through layered expertise while maintaining Superpowers’ workflow discipline and your optimization goals. If specific skills prove problematic in practice, you can revert just those to standalone status while keeping the rest integrated.
