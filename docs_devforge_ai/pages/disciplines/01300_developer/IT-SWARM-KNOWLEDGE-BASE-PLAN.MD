## Filing Strategy

### 1. Human Documentation (Source of Truth)

**Location**: `docs/procedures/`, `docs/standards/`

**Purpose**: Human-readable procedures and standards

**Contents**:
- Page implementation procedures
- UI/UX standards
- File naming conventions
- Accordion integration rules

**Access Pattern**: Reference only - agents read and parse

### 2. Generated System State

**Location**: `docs/schema/reports/`

**Purpose**: Machine-readable system state snapshots

**Generation**: Via scripts in `docs/schema/code/`

**Contents**:
- `index-buttons-modals.md` - Button/modal inventory
- `index-deep-agents.md` - Agent registry
- `index-pages.md` - Page structure
- `current-full-schema.md` - Database schema

**Access Pattern**: Read-only - regenerated by scripts

### 3. Agent-Embedded Rules

**Location**: `deep-agents/deep_agents/agents/shared/it_specialists/`

**Purpose**: Critical rules embedded in agent code for immediate enforcement

**Implementation**:
```python
# In 0040_agent_validator.py or 0000_base_it_specialist.py

PAGE_IMPLEMENTATION_RULES = {
    "background_image_policy": {
        "allowed_page_types": ["LANDING_PAGES", "ADMIN_DASHBOARDS"],
        "forbidden_page_types": [
            "DATA_MANAGEMENT",
            "WORKFLOW_PROCESSING",
            "FORM_ONBOARDING",
            "DOCUMENT_PROCESSING",
            "TIME_TRACKING"
        ]
    },
    "mandatory_components": [
        "AccordionProvider",
        "logout_button",
        "settings_manager_initialization"
    ],
    "state_button_pattern": {
        "initialize_state": None,  # NOT "agents"
        "default_return": "[]"     # Empty array, not agentsButtons
    },
    "template_a_required": True,
    "alphabetical_link_ordering": True
}

VALIDATION_CHECKLISTS = {
    "page_creation": [
        "Template A CSS imported",
        "AccordionProvider wrapped",
        "Logout button present",
        "Settings manager initialized",
        "Background policy followed"
    ]
}
```

## Implementation Phases

### Phase 1: Immediate (Week 1-2)

**Goal**: Enable basic validation with embedded rules

**Tasks**:
1. Extract critical rules from `0000_PAGE_IMPLEMENTATION_PROCEDURE.md`
2. Embed rules in `0000_base_it_specialist.py`
3. Add validation methods to `0040_agent_validator.py`
4. Document reference paths in agent code

**Deliverables**:
- Embedded PAGE_IMPLEMENTATION_RULES
- Basic validation methods
- Documentation references

### Phase 2: System State Integration (Week 3-4)

**Goal**: Integrate schema reports for system-aware validation

**Tasks**:
1. Add methods to read `docs/schema/reports/` files
2. Parse button/modal indices for validation context
3. Parse agent indices for agent creation context
4. Add schema state awareness to validation logic

**Deliverables**:
- Schema report parsing methods
- System state awareness in agents
- Validation with context

### Phase 3: Skill Integration (Week 5-8)

**Goal**: Integrate with MCP skills as they mature

**Tasks**:
1. Add MCP client capability to base agent class
2. Implement skill call wrappers
3. Add fallback to embedded rules
4. Test skill integration

**Deliverables**:
- MCP client integration
- Skill call methods
- Fallback mechanisms

### Phase 4: Automation (Week 9-12)

**Goal**: Automate knowledge base synchronization

**Tasks**:
1. Create knowledge sync agent (0070_knowledge_sync_specialist.py)
2. Implement automatic procedure → rule extraction
3. Add schema regeneration triggers
4. Implement drift detection

**Deliverables**:
- Knowledge sync specialist
- Automated rule updates
- Schema regeneration automation

## Validation Rules Summary

### Critical Rules from Page Implementation Procedure

1. **Background Image Policy**
   - ONLY 2 page types allowed: LANDING_PAGES, ADMIN_DASHBOARDS
   - All others: NO backgrounds

2. **Mandatory Components**
   - AccordionProvider wrapper
   - Logout button
   - Settings manager initialization

3. **State Button Pattern**
   - Initialize state to `null` (not "agents")
   - Default return `[]` (empty array)

4. **Template A Compliance**
   - Import template-a-standard.css
   - Use CSS variables
   - Card-based layout

5. **Accordion Integration**
   - 3-tier fallback system
   - Alphabetical link ordering
   - Server template + client fallback
