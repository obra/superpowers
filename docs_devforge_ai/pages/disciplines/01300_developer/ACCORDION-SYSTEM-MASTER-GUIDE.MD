## Discipline Parameter URL Access Methodology (November 2025)

### URL-Based Navigation for Discipline-Specific Page Access
A parameter-based URL system has been implemented to enable discipline-specific navigation links within the accordion system. This allows users to access the same shared page with different filtering behavior based on their entry point.

#### Navigation Implementation
Accordion section links use discipline parameters to signal specialized access:

```javascript
// Server-side accordion configuration (MASTER_TEMPLATE)
{
  title: 'Template & Form Management',
  url: '/templates-forms-management?discipline=01900' // Procurement access
},
{
  title: 'Template & Form Management',
  url: '/templates-forms-management?discipline=02400' // Safety access
}
// Governance section uses default URL without parameter
```

#### Client-Side Parameter Detection
Pages detect discipline parameters for role-based filtering:

```javascript
// Client-side URL parameter detection
const urlParams = new URLSearchParams(window.location.search);
const disciplineParam = urlParams.get('discipline');

// Role assignment and filtering logic
if (disciplineParam === '01900') {
  detectedRole = 'discipline'; // Procurement templates only
} else if (disciplineParam === '02400') {
  detectedRole = 'discipline'; // Safety templates only
} else {
  // Default governance access - all templates
}
```

#### Navigation Access Matrix
| Accordion Section | URL Pattern | User Access Level | Content Filter |
|------------------|-------------|-------------------|----------------|
| **Procurement Section** | `?discipline=01900` | Discipline-limited | Procurement templates only |
| **Safety Section** | `?discipline=02400` | Discipline-limited | Safety templates only |
| **Governance Section** | *(no parameter)* | Full access | All templates from all disciplines |

#### Implementation Requirements
- **Parameter Propagation**: URL parameters maintained through navigation
- **Role Integration**: Parameter-derived roles fed into existing permission systems
- **Security Consistency**: Works within established authentication frameworks
- **Fallback Support**: Graceful handling of invalid parameter combinations

#### Usage Examples
**Procurement Workflow Access**:
- **Link**: Procurement accordion → "Template & Form Management"
- **URL**: `/templates-forms-management?discipline=01900`
- **Result**: User sees only procurement-related templates (01900 codes)

**Safety Management Access**:
- **Link**: Safety accordion → "Template & Form Management" 
- **URL**: `/templates-forms-management?discipline=02400`
- **Result**: User sees only safety-related templates (02400 codes)

**Governance Oversight**:
- **Link**: Governance accordion → "Template & Form Management"
- **URL**: `/templates-forms-management` (no parameter)
- **Result**: Full governance access to all discipline templates

#### Integration Benefits
- **Same Page Architecture**: Single templates management page serves all disciplines
- **Role-Based Filtering**: Dynamic content based on navigation source
- **Security Compliance**: Parameter system integrates with existing access controls
- **Scalable Design**: Easy to extend to additional disciplines

This methodology enables shared page components to provide tailored experiences while maintaining consistent navigation patterns and security controls throughout the accordion system.
