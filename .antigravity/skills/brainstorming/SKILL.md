# Brainstorming Ideas Into Designs

> **This skill mirrors the `/brainstorming` workflow.**

## Overview
Turn ideas into fully formed designs through collaborative dialogue. Understand context, ask questions, propose approaches, present design, get approval.

**HARD GATE:** Do NOT write any code until you have presented a design and the user has approved it.

## Checklist
1. **Explore project context** — check files, docs, recent commits
2. **Ask clarifying questions** — one at a time
3. **Propose 2-3 approaches** — with trade-offs and recommendation
4. **Present design** — in sections, get approval after each
5. **Write design doc** — save and commit
6. **Transition** — use `writing-plans` skill to create implementation plan

## Key Principles
- One question at a time
- Multiple choice preferred
- YAGNI ruthlessly
- Explore alternatives before settling
- Incremental validation
- Scale sections to complexity
